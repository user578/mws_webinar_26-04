## Этап 1: Подготовка окружения

### 1.1. Локальные утилиты (Linux / macOS)

Убедитесь, что у вас установлены следующие инструменты:

*   **Terraform** (версия >= 1.5): [Установка](https://developer.hashicorp.com/terraform/downloads)
*   **Kubectl**: [Установка](https://kubernetes.io/docs/tasks/tools/)
*   **Helm**: [Установка](https://helm.sh/docs/intro/install/)
*   **kubectx / kubens**: [Установка](https://github.com/ahmetb/kubectx) (полезно для переключения контекстов)
*   **MWS CLI**: [Установка](https://mws.ru/docs/cloud-platform/mws-cli/)
*   **jq** — консольный обработчик JSON (нужен для скриптов).

### 1.2. Подготовка в консоли MWS (Manual)

Перед запуском автоматизации выполните несколько ручных шагов:

1.  **Создайте проект**: Зайдите в [консоль MWS](https://console.mws.ru/) и создайте проект с именем `webinar-dev`. Запомните его `Project ID`.
2.  **Создайте Сервисный Аккаунт (SA) для Terraform**:
    *   Перейдите в раздел **IAM** -> **Service Accounts**.
    *   Создайте SA `terraform-sa` с ролью `editor` на проект.
    *   Создайте для него **Authorized Key** (JSON формат).
    *   Сохраните файл как [secrets/terraform_sa_key.json](secrets/terraform_sa_key.json) в корне этого проекта.
3.  **Создайте SA для узлов кластера**:
    *   Создайте SA `k8s-node-sa` (роли не требуются, если не планируете специфических интеграций, но для MK8S обычно нужен SA).

## Этап 2: Развертывание инфраструктуры

Мы используем Terraform для создания сети (VPC), подсетей и управляемого кластера Kubernetes (MK8S).

1.  **Настройте переменные**: Откройте [tf/variables.tf](tf/variables.tf) или [tf/providers.tf](tf/providers.tf) и укажите ваш `project_id`.
2.  **Проверьте план**:
    ```bash
    ./00_plan_cloud_env.sh
    ```
    Этот скрипт инициализирует Terraform и покажет, какие ресурсы будут созданы.
3.  **Разверните окружение**:
    ```bash
    ./10_prepare_cloud_env.sh
    ```
    **Что произойдет:**
    *   Создадутся VPC и подсети.
    *   Развернется кластер MK8S.
    *   Скрипт автоматически настроит `mws cli`, получит `kubeconfig` и сохранит его в [secrets/config](secrets/config).
    *   Проверит доступность узлов кластера через `kubectl`.

## Этап 3: Сеть и Балансировка

Чтобы приложение было доступно из интернета, нам нужен Ingress-контроллер и SSL.

1.  **Установите Nginx Ingress Controller**:
    ```bash
    ./20_deploy_ingress_nginx.sh
    ```
    Скрипт создаст облачный Network Load Balancer и привяжет его к Ingress.
2.  **Установите сертификаты**:
    ```bash
    ./21_install_tls_certs.sh
    ```
    Мы используем предустановленные сертификаты для домена вебинара. В реальном проекте здесь может быть интеграция с `cert-manager`.

## Этап 4: GitLab и CI/CD Runner

Для управления кодом и запуска пайплайнов мы развернем GitLab прямо внутри кластера.

1.  **Установите GitLab**:
    ```bash
    ./22_deploy_gitlab.sh
    ```
    GitLab будет доступен по адресу, указанному в скрипте (например, `https://gitlab.webinar-26-04.nufl.ru`).
2.  **Установите GitLab Runner**:
    ```bash
    ./23_deploy_runner.sh
    ```
    Скрипт автоматически извлечет токен регистрации из GitLab и зарегистрирует раннер в кластере.

## Этап 5: Artifact Registry

Artifact Registry — это место, где будут храниться собранные Docker-образы вашего приложения.

1.  **Создайте реестр (Manual)**:
    *   В консоли MWS перейдите в **Artifact Registry**.
    *   Создайте новый реестр (например, `webinar-registry`).
2.  **Создайте SA для Registry**:
    *   Создайте SA `registry-sa` с ролью `registry.pusher` (для сборки) и `container-registry.puller` (для деплоя).
    *   Создайте API-ключ для этого SA.
3.  **Настройте секреты в кластере**:
    *   Сохраните ваш API-ключ в файл [secrets/registry_password](secrets/registry_password).
    *   Выполните:
    ```bash
    source env.sh
    ./24_prepare_app_ns.sh
    ```
    Этот скрипт создаст namespace `app` и добавит `imagePullSecrets`, чтобы Kubernetes мог скачивать ваши приватные образы.

## Этап 6: Запуск разработки

Теперь всё готово для деплоя самого приложения!

1.  **Приложение ([/app](app/))**: Это простое Go-приложение "Cool Business App", которое визуализирует "курс доллара" в зависимости от "очередей в кофейнях".
    *   [main.go](app/main.go) — бэкенд и фронтенд (HTML/JS).
    *   [Dockerfile](app/Dockerfile) — инструкция по сборке.
    *   [chart/](app/chart/) — Helm-чарт для деплоя в K8s.
2.  **Запуск пайплайна**:
    *   Загрузите код в созданный GitLab.
    *   Пайплайн ([.gitlab-ci.yml](app/.gitlab-ci.yml) — должен быть в корне репозитория приложения) соберет образ с помощью **Kaniko** (dockerless сборка) и задеплоит его через Helm.

3.  **Проверка результата**:
    ```bash
    ./30_check_result.sh
    ```
    Скрипт покажет статус подов и URL вашего приложения.

## Вопросы безопасности

*   **Принцип наименьших привилегий**: Для каждого этапа мы используем отдельные Service Accounts.
*   **Секреты**: Все ключи и конфиги вынесены в директорию `secrets/` (которая должна быть в `.gitignore`).
*   **Сетевая безопасность**: Группы безопасности (Firewall) в Terraform настроены только на необходимые порты (80, 443, 6443).

Чтобы удалить все созданные ресурсы и не тратить бюджет:
```bash
cd tf
```
